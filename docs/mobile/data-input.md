# Data Input & Tracking System

The System needs data to function. This document defines how players input their fitness data, what integrations we support, and how we handle verification.

---

## Philosophy

**The System trusts, then verifies when possible.**

We're not building a lie detector. Players who cheat are only cheating themselves. But we make honest tracking as frictionless as possible by:

1. Automating what can be automated (steps, sleep)
2. Simplifying what can't (workouts, protein)
3. Never requiring burdensome logging
4. Trusting self-reports when necessary

---

## Data Source Hierarchy

For each metric, we prefer sources in this order:

```
1. Automatic (Health APIs) â€” Most reliable, zero effort
2. Semi-automatic (Imported) â€” Reliable, minimal effort
3. Manual (User input) â€” Required fallback
```

## Verification Levels

Not all data has the same trust level. We track this:

| Level | Description | Example |
|-------|-------------|---------|
| **Verified** | Hardware-confirmed | Steps from HealthKit |
| **Imported** | From trusted source | Workout from fitness app |
| **Self-Reported** | User claims it | "Yes I hit protein" |
| **Estimated** | System inferred | Wake time from phone unlock |

**Why Track This?**

1. Leaderboard integrity (verified data ranked higher in ties)
2. Pattern analysis (are self-reports consistent?)
3. Future features (verified challenges, competitions)

## Privacy & Data Handling

**What We Store**:
- Daily totals (steps, workout yes/no, protein yes/no)
- Timestamps (wake time, workout time)
- Source of each data point
- Correction history

**What We Don't Store**:
- Detailed health records
- Location data (unless nature quest opted-in)
- Heart rate, weight, or other sensitive metrics
- Third-party app raw data

**Narrative**:
```
DATA HANDLING NOTICE

The System records:

â€¢ Timestamps
â€¢ Data sources
```
